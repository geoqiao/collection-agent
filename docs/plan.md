# 计划：Skill 优化 + 前端可视化

## 目标

1. **Skill 优化**：重写所有 skill prompt，基于真实催收谈判技巧，核心原则——
   - **不给减免**，只给延期/分期
   - 施压递增：温和 → 紧迫 → 警告

## Phase 1: Skill 重写（调研 + 优化）

### 调研结论

真实催收谈判的核心技巧（合法合规）：

1. **话术公式**：事实陈述 + 法律依据 + 损失量化 + 解决方案
2. **时间锚定**：每次对话设定明确 deadline（"今天几点前""明天中午前"）
3. **损失厌恶**：强调不还款会失去什么（征信、诉讼费、子女教育）
4. **小承诺策略**：先让用户答应一个小行动（"今天先还 200""明天回电话"）
5. **共情但不妥协**："我理解您的困难，但账单必须处理"
6. **阶梯施压**：
   - Round 1（逾期 1-7 天）：温和提醒，了解原因
   - Round 2（逾期 8-30 天）：紧迫施压，告知后果
   - Round 3（逾期 30+ 天）：警告升级，提及法律程序
7. **书面留痕**：所有还款承诺必须确认具体时间、金额

### Skill 重写清单

| Skill | 核心变化 |
|---|---|
| `onboard` | 首次联系建立权威，告知后果，不给协商空间 |
| `negotiation` | **不给减免**，只给延期/分期；要求书面承诺；时间锚定 |
| `reengage` | 沉默后施压递增，引用之前的承诺 |
| `complaint` | 安抚但强调账单义务，不给减免承诺 |
| `crisis` | 保持安全优先，暂停但不放弃 |
| `stop` | 确认 DNC，但提醒法律义务仍在 |
| `standard` | 敏感职业标准提醒，无协商内容 |
| `followup` | 跟进承诺，强调违约后果 |
| `troubleshoot` | 协助还款操作 |
| `payment_guidance` | 引导完成还款，确认到账 |

## Phase 2: 前端页面

### 技术栈
- **后端**：FastAPI（新增路由 + WebSocket）
- **前端**：纯 HTML + Tailwind CSS + Vanilla JS
- **通信**：WebSocket（实时推送 LLM thinking + 状态变更）

### 页面结构
```
/ (首页)
├── 左侧：配置面板
│   ├── 用户ID
│   ├── 姓名
│   ├── 逾期天数
│   ├── 金额
│   ├── 职业
│   └── [创建用户] 按钮
├── 中间：对话面板
│   ├── 消息历史（用户/机器人）
│   ├── 输入框
│   ├── [发送] / [触发催收] / [模拟支付]
│   └── 当前状态显示
└── 右侧：调试面板
    ├── 当前意图
    ├── 会话状态
    ├── 协商轮数
    ├── Skill 名称
    └── Thinking 过程（可折叠）
```

### API 设计
```
POST /api/users          → 创建用户
GET  /api/users/{id}     → 获取用户状态
POST /api/events         → 发送事件
WS   /ws/{user_id}       → WebSocket 实时推送
```
