用户交互事件处理流程

1

合规时间检查（Layer 1）

检查当前是否在 8:00-20:00 合规窗口内。outreach 事件在外部时间被拦截。

2

One-Way Door 检查

如果 session 处于 ESCALATED/STOPPED/CRISIS/DISPUTED 状态，直接返回固定模板，绕过 LLM。

3

意图识别

LLMIntentRecognizer 通过 CoT + XML 结构化输出识别用户意图（A/B/C/D/E/STOP/CRISIS）。

4

Skill 选择

SkillRegistry 根据意图匹配 Skill。无匹配时 fallback 到 onboard。

5

Skill 执行（ReAct 循环）

SkillExecutor 驱动 LLM 通过 ReAct 循环：观察 → 思考 → 行动（reply/tool_call/escalate/end）。

6

结果处理

更新状态机、保存用户状态、返回响应文本。

3 Skill 系统

所有 Skill 是纯配置类，继承自 Skill 基类（dataclass），不包含任何 execute() 方法。

关键约束：grep -rn 'async def execute' src/skills/*.py 只返回 executor.py。Skill 类只有配置字段。

10 个催收 Skills

Skill 触发条件 One-Way Door 核心策略
OnboardSkill SCHEDULED_OUTREACH / USER_LOGIN 否 首触施压，直接说明逾期情况和紧迫性
PaymentGuidanceSkill WILLING_TO_PAY 否 催促立即还款，不给拖延空间
NegotiationSkill UNWILLING_TO_PAY / 协商请求 否 先施压全额，减免是最后手段
ReEngageSkill SILENCE_TIMEOUT / 回避 否 再次施压，强调回避只会升级

ComplaintHandlingSkill COMPLAINT / 投诉关键词 是 固定道歉模板 → 暂停催收 → 转人工
DisputeResolutionSkill DISPUTE / 争议关键词 是 固定模板 → 暂停催收 → 转人工
CrisisInterventionSkill CRISIS / 危机关键词 是 固定模板 → 暂停催收 → 转人工

stateDiagram-v2 [*] --> NORMAL NORMAL --> PENDING_ESCALATE: 用户要求协商 NORMAL --> ESCALATED: 投诉/威胁检测 NORMAL --> STOPPED: STOP 关键词 NORMAL --> CRISIS: 危机关键词 NORMAL --> DISPUTED: 争议提出 NORMAL --> RESOLVED: 还款成功 PENDING_ESCALATE --> NORMAL: 用户改变主意 PENDING_ESCALATE --> ESCALATED: 用户变 hostile PENDING_ESCALATE --> STOPPED: 用户要求停止 PENDING_ESCALATE --> RESOLVED: 达成协议 ESCALATED --> RESOLVED: 人工标记解决 STOPPED --> RESOLVED: 人工标记解决 CRISIS --> RESOLVED: 人工标记解决 DISPUTED --> RESOLVED: 人工标记解决 note right of NORMAL Flowing 状态 每轮可自由切换意图 end note note right of ESCALATED One-Way Door 固定模板，绕过 LLM end note
Flowing 状态

NORMAL：正常催收，每轮重新识别意图

PENDING_ESCALATE：协商中，可能resolve或escalate

One-Way Door

ESCALATED：投诉/威胁，固定道歉模板

STOPPED：用户要求停止，确认后不再联系

CRISIS：危机关键词，安慰+转人工

DISPUTED：账单争议，暂停催收

意图分类

类别 含义 对应 Skill
A 合作：愿意还款或询问还款方式 PaymentGuidanceSkill
B 协商：表示困难，希望延期或分期 NegotiationSkill
C 回避：回避问题，不愿深入讨论 ReEngageSkill
D 争议：质疑账单真实性或金额 DisputeResolutionSkill（One-Way Door）
E 投诉/威胁：表达不满，威胁投诉 ComplaintHandlingSkill（One-Way Door）
STOP 停止联系：明确要求停止催收 StopHandlingSkill（One-Way Door）
CRISIS 危机：提及自杀、重病、极端困难 CrisisInterventionSkill（One-Way Door）

def estimate_tokens(self, text: str) -> int:
    """CJK-aware token approximation."""
    count = 0
    for ch in text:
        if ('一' <= ch <= '鿿' or
            '㐀' <= ch <= '䶿'):
            count += 2  # CJK
        else:
            count += 1  # ASCII / Latin
    return count

12 渠道与 XSS 防护

所有渠道在 send() 方法中调用 escape_output()，防止 LLM 生成内容中的 XSS 攻击。

# src/channels/_escape.py
import html

def escape_output(text: str) -> str:
    if not text:
        return text
    return html.escape(text, quote=True)  # escapes &, <, >, ", '
