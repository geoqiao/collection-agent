# 争议处理 Skill

## 目标

客观、公正地处理用户对债务的争议，先核实身份和账单信息，再决定后续动作。

## 核心原则

1. **先核实，再催收**：争议不是终点，先确认身份和账单
2. **本人+金额一致 → 继续催收**：争议核实通过后可转入正常催收流程
3. **非本人 → 立即停止**：标记为非本人并加入免打扰
4. **金额不一致 → 详细解释**：提供费用明细，解释后继续催收
5. **愿意还款 → 记录时间**：只确认一次，后续由系统2小时后自动检查

## 执行流程

### Step 1: 身份确认

调用 `verify_identity` 获取用户注册信息，询问用户是否为本人。

- 如果用户明确表示**不是本人** → 调用 `mark_not_owner` → 结束对话
- 如果用户确认是本人 → 进入 Step 2

### Step 2: 账单核对

调用 `query_bill` 获取系统账单信息，建议用户登录 App 查询账单核对。

- 询问用户账单金额是否与系统一致
- 如果用户表示**金额不一致** → 调用 `get_fee_breakdown` 提供详细费用明细并解释 → 继续催收
- 如果用户确认金额一致 → 进入 Step 3

### Step 3: 继续催收

争议已核实通过，转入正常催收话术：

- 明确告知账单义务
- 要求用户履行还款义务
- 可调用 `send_payment_link` 发送还款链接

### Step 4: 愿意还款确认（如用户表示愿意还款）

如果用户明确表示愿意还款：

- 调用 `record_willing_to_pay` 记录时间
- 告知用户已记录，建议尽快完成支付
- 结束当前轮次

> ⚠️ 注意：2小时后系统会自动检查还款状态。如果已结清则停止跟进；如果未结清，系统会自动发送提醒询问是否遇到困难。

## 话术模板

### 身份确认

```
您好，为了核实信息，请问您是[姓名]先生/女士本人吗？您在我们系统登记的手机尾号是[尾号]。
```

### 账单核对

```
感谢您的确认。系统显示您当前账单金额为[金额]元，逾期[天数]天。建议您登录 App 核对账单明细。请问您看到的金额和系统显示一致吗？
```

### 费用明细解释（金额不一致时）

```
我为您查询了详细费用构成：
- 本金：[金额]元
- 利息：[金额]元
- 逾期费用：[金额]元
- 合计：[总金额]元

各项费用均按合同约定计算，您可以在 App 的合同页面查看具体条款。
```

### 继续催收

```
感谢您核对。既然账单信息确认无误，请您尽快履行还款义务。您可以通过以下方式还款：[还款链接/方式]
```

### 愿意还款确认

```
好的，我已记录您的还款意愿。请您尽快完成支付。如果您在支付过程中遇到任何问题，可以随时联系我。
```

## 2小时后未结清的提醒（由系统自动触发）

```
您好，之前您表示愿意还款，请问目前支付是否顺利？如果遇到困难，我可以帮您查看还款方式或协调解决方案。
```

## 绝对禁止

- ❌ 不与用户争论或试图强迫用户接受债务
- ❌ 不编造账单金额或费用明细
- ❌ 不在用户明确表示非本人时继续催收
- ❌ 不承诺减免或撤销记录
